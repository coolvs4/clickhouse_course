**Область применения и первое представление**

Примеры использования ClickHouse в компаниях на рынке:

1. ***Cloudflare***
   Решалась задача предоставления клиентам аналитики HTTP-трафика. Кол-во сообщений в теме Kafka для HTTP-запросов - 6 млн в секунду.

   ClickHouse отлично подошел для решения этой задачи, поскольку необходимо анализировать множество различных показателей по каждому запросу. Колоночная схема хранения позволяет эффективно извлекать данные из широких таблиц с большим количеством столбцов.

   Для анализа данных необходимо создание различных сводок по временным периодам с различной временной детализацией. ClickHouse позволяет решить эту задачу с использованием материализованных представлений.

   В качестве источника данных до использования ClickHouse применялась Kafka. ClickHouse отлично интегрируется в эту схему, благодаря наличику множества движков для интеграции с внешними системами, в т.ч. Kafka.

   Также важным требованием было наличие отказоустойчивости и линейного масштабирования - и снова ClickHouse, позволяющий строить различные конфигурации кластеров с шардированием и репликацией.
2. ***SigNoz*** (инструмент, который помогает разработчикам и DevOps-командам следить за состоянием приложений, быстрее находить и устранять проблемы)

   ClickHouse выступает в роли основного хранилища данных для платформы SigNoz. Проще говоря: SigNoz собирает телеметрию (метрики, трейсы, логи из ваших сервисов), а ClickHouse её надёжно и быстро сохраняет.

   ClickHouse предоставляет расширенный набор аналитических функций (агрегатные, оконные, для работы с временными рядами), что позволяет выполнять в платформе сложные аналитические запросы прямо в базе. Это даёт пользователям больше гибкости при исследовании данных и построении дашбордов.

   ClickHouse использует сжатие данных и партиционирование. Это позволяет существенно сократить объём занимаемой памяти и, как следствие, снизить затраты на хранение данных при сохранении высокой производительности при запросах.
